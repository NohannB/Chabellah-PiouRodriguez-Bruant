## Créer un *fork* de ce dépôt (1 par équipe seulement)

Vous devez créer une copie (*fork*) de ce dépôt puis vous en servir comme dépôt pour votre projet. Pour cela :
1. Cliquez sur le bouton `Fork` en haut à droite (ou allez sur [ce lien](https://github.com/ENSG-POO-2026-ING1/Projet-ING1/fork))
2. Dans le menu déroulant "Owner", choisissez "ENSG-POO-2026-ING1" (pour créer le dépôt dans l'organisation et non dans votre compte personnel)
3. Dans le champ "Repository name", entrez vos noms de famille (exemple, pour nous : "Nghien-Meynard-Spur-Szanto")
4. La description n'est pas obligatoire et pourra être modifiée plus tard
5. Vous pouvez ignorer l'option "Copy the `main` branch only" (de toute manière, il n'y a qu'une seule branche sur ce dépôt)
6. Validez en cliquant sur le bouton vert "Create fork" en bas à droite
7. Vous arrivez sur votre dépôt, qui est une copie de celui-ci, mais où vous avez tous les accès

## Ajouter ses coéquipiers au dépôt

Après avoir réalisé un *fork* pour votre équipe, vous devez accorder l'accès à vos coéquipiers. Pour cela :
1. Cliquez sur le bouton `Settings` en haut de votre dépôt
2. Sur la gauche, cliquez sur `Collaborators and teams`
3. Si GitHub vous demande de confirmer votre identité à l'aide de la double authentification, procédez à la vérification en suivant les étapes
4. Parmi tous les boutons, cherchez `Add people` (à côté de `Add teams`, vers la droite de la page), et cliquez dessus
5. Dans le menu déroulant qui apparait, choisissez un membre de votre équipe (à partir de son identifiant GitHub)
6. Sélectionnez le rôle "Admin" pour donner un accès total à vos coéquipiers
7. Validez en cliquant sur le bouton vert "Add selection"
8. Répétez l'opération pour chaque membre de l'équipe
9. Vous arrivez à nouveau sur la page qui récapitule les accès, vous devriez y voir vsos coéquipiers dans l'onglet "Direct access" (ainsi que l'équipe "Teachers" dans l'onglet "Organization access", très important pour qu'on puisse accéder à votre projet !)

## *Cloner* le dépôt sur vos machines

Une fois que vous avez tous accès au dépôt de votre équipe, vous pouver le récupérer (*clone*) sur vos machines pour travailler en local. Vous ferez ensuite un *commit* et *push* pour synchroniser vos modifications avec le serveur GitHub. Référez-vous à votre cours sur l'utilisation de Git pour cela. En cas de problème, n'hésitez pas à solliciter un enseignant.

> [!WARNING]
> L'intérêt de Git (et GitHub) est de conserver son travail, avec les différentes versions et étapes d'avancement du projet, et de le synchroniser entre tous les membres. N'hésitez surtout pas à créer des copies sur des branches annexes pour conserver une trace de vos tests, prototypes, échecs, ou fonctionnalités abandonnées. Vous n'avez pas de limite de stockage (du moins, elle est suffisamment haute pour que vous n'ayez pas à vous en soucier) ou de nombre de branches, alors profitez-en. L'an dernier, un groupe a perdu plusieurs heures de travail dans une mauvaise manipulation, ce qui a impacté leur note finale. Ne faites pas la même erreur, et gardez des sauvegardes avant d'effectuer des opérations dont vous n'êtes pas certain du résultat !

## Personnaliser son dépôt et son `README.md`

Dorénavant, il s'agit du dépôt de votre équipe, vous pouvez le personnaliser comme bon vous semble. N'hésitez pas à vous l'approprier, et surtout à modifier ce fichier `README.md`, pour vous en servir comme documentation de base sur votre projet.
